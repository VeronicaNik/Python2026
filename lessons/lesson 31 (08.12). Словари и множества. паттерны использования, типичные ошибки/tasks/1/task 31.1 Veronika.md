# Задание 31.1. Словари и множества. паттерны использования, типичные ошибки
## Блок урока
Словарь как структура быстрого доступа — Блок показывает, как превращать повторяющиеся данные в словарь и мгновенно находить значения по ключу.

## Условие задачи
Турагентство отслеживает, кто последний обновлял описание тура. Менеджер проверяет, на кого можно положиться при срочном апдейте. Создай словарь editor, где ключ — название тура, а значение — имя ответственного менеджера. Затем выведи информацию для переменных request_tour и new_offer.

## Пример входных данных
```python
tours = [
    ('Iceland', 'Лена'),
    ('Georgia', 'Тимофей'),
    ('Mongolia', 'Аделина'),
]
request_tour = 'Georgia'
new_offer = 'Chile'
```

## Пример выходных данных
```text
Georgia -> Тимофей
Chile -> не занесён
```

## Критерии успешного выполнения
- Список пар преобразован в словарь editor с ключом 'название тура'.
- Для обращения к словарю используется метод get с значением по умолчанию.
- Выведены ответы для request_tour и new_offer в заданном формате.
- Код не падает при запросе отсутствующего ключа.

## Подсказки
- Пройдись по списку tours в цикле и наполняй словарь editor.
- Используй множественное присваивание: key, value = запись из tours.
- Для отсутствующих данных возвращай строку 'не занесён'.

## Частые ошибки и ограничения
- Не обращайся к словарю через квадратные скобки, если нет проверки на существование ключа.
- Не оставляй ключи кортежами целиком — нужен именно идентификатор.
- Не дублируй вывод: достаточно одной строки на каждый запрос.
