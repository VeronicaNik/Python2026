# Задание 29.3. Коллекции уровня «джун+». списки, срезы, вложенные структуры

## Блок урока
Мутация и методы списков. Учимся добавлять, вставлять и удалять элементы так, чтобы быстро адаптировать расписание без копирования данных.

## Условие задачи
Ты планируешь серию мини интервью для документального проекта:
1. Создай список гостей по порядку съемок.
2. Добавь через `extend` двух экспертов, которых подтвердили в последний момент.
3. Вставь ознакомительный звонок в позицию №2, чтобы он шел сразу после открытия.
4. Удали по значению гостя, который отказался.
5. Перемести последний элемент в начало, используя `pop` и `insert(0, ...)`.
6. Выведи итоговый список с нумерацией и темой каждого интервью.

## Пример входных данных
```python
lineup = ["открытие. режиссер", "история проекта. продюсер", "опыт героев", "закрытие"]
late_guests = ["техника. оператор", "музыка. композитор"]
```

## Пример выходных данных
```
1. музыка. композитор
2. открытие. режиссер
3. ознакомительный звонок
4. история проекта. продюсер
5. опыт героев
6. техника. оператор
```

## Критерии успешного выполнения
- Использованы минимум три разных метода изменения списка
- Вставка, удаление и перемещение выполняются корректно
- Итоговый список распечатан после всех операций
- В выводе указано, зачем нужен каждый пункт

## Подсказки
- После `extend` новые элементы окажутся в конце `lineup`
- Индекс 1 вставит элемент как второй по счету
- `pop()` без аргумента берет последний элемент. его можно сразу вставить в начало через `insert(0, item)`

## Частые ошибки и ограничения
- Не удаляй элемент, если не проверил его наличие
- Не меняй список в том же цикле, где его перебираешь
- Следи, чтобы индексы не выходили за пределы
- Работай с исходным списком, не создавай копии
