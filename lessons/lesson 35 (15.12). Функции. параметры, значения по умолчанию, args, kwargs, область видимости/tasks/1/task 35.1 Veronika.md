# Практическое задание 35.1

Это задание относится к первому блоку темы "Функции". Вы уже знакомы с коллекциями (списки, словари) и базовыми алгоритмами. Сейчас мы начинаем изучать функции — основной инструмент для структурирования кода. В этом блоке вы научитесь объявлять функции с помощью `def`, передавать в них аргументы (параметры) и возвращать результат с помощью `return`. Это фундамент для всех последующих тем о функциях.

## Задания

### Описание задачи
Вероника, ваша задача — создать простую функцию для расчета стоимости покупок в магазине. У вас есть список товаров, где каждый товар представлен словарем с ключами `"название"` и `"цена"`. Вам нужно написать функцию `calculate_total`, которая принимает этот список и возвращает общую сумму всех покупок.

**Контекст:** Вы помогаете автоматизировать работу небольшого магазина. Кассир вводит товары в систему, и программа должна быстро подсчитывать итог. Функция — это идеальный способ вынести логику подсчета в отдельный, переиспользуемый блок кода.

### Пример входных и выходных данных
**Пример 1:**
```python
# Входные данные (список словарей)
покупки = [
    {"название": "Молоко", "цена": 90},
    {"название": "Хлеб", "цена": 50},
    {"название": "Яблоки", "цена": 120}
]

# Вызов функции и вывод результата
итог = calculate_total(покупки)
print(итог)  # Ожидаемый вывод: 260
```

**Пример 2:**
```python
# Входные данные (список с одним товаром)
покупки = [
    {"название": "Книга", "цена": 500}
]

итог = calculate_total(покупки)
print(итог)  # Ожидаемый вывод: 500
```

**Пример 3:**
```python
# Входные данные (пустой список)
покупки = []

итог = calculate_total(покупки)
print(итог)  # Ожидаемый вывод: 0
```

### Критерии проверки и ограничения
1.  Функция должна называться `calculate_total`.
2.  Функция должна принимать **ровно один параметр** (список покупок).
3.  Функция должна **возвращать** (`return`) общую сумму (целое число). Не используйте `print` внутри функции для вывода результата.
4.  Если список пустой, функция должна вернуть `0`.
5.  Для решения задачи используйте только пройденные темы: `def`, параметр, аргумент, `return`. Не используйте значения по умолчанию, `*args`, `**kwargs` или другие темы из будущих блоков.
6.  Внутри функции вам понадобится цикл `for` для перебора списка и сложение чисел — это вы уже умеете.

### Решение задачи (псевдокод)
1.  Объяви функцию с именем `calculate_total`. Укажи один параметр, например, `items_list`.
2.  Создай внутри функции переменную-счетчик (например, `total_sum`) и установи ее начальное значение в `0`.
3.  Начни цикл, который будет перебирать каждый элемент (словарь) в переданном списке `items_list`.
4.  Для каждого элемента (словаря) получи значение по ключу `"цена"`.
5.  Прибавь это значение к переменной-счетчику `total_sum`.
6.  После завершения цикла, верни (`return`) значение переменной `total_sum` как результат работы функции.

### Рекомендации
-   **Вынести в отдельную функцию:** Да, вся логика подсчета суммы должна быть инкапсулирована внутри функции `calculate_total`. Это делает код чище и позволяет повторно использовать подсчет в разных частях программы.
-   **PEP8:**
    -   Используйте осмысленные имена для функции и параметра. `calculate_total` и `items_list` — хорошие примеры.
    -   После объявления функции `def` оставляйте две пустые строки перед следующим блоком кода (например, перед вызовом функции).
